Om CSS

Vad är CSS?

CSS är en förkortning av Cascading Style Sheet och utgör en grundläggande byggsten för webben, tillsammans med HTML och Javascript. Det är ett språk som används för att styra utseende och definiera layouten av en webbsida. CSS behöver ett märkspråk som HTML för att kunna utföra någonting på en webbsida. CSS kan alltså inte åstadkomma någon effekt på egen hand.

Varför finns CSS?

HTML har som syfte att definiera strukturen och innehållet på en webbsida. När en webbläsare renderar ett HTML-dokument översätts det till en visuellt begriplig form, och för att presentera ett läsbart innehåll tillämpas fördefinierade stilar, som bland annat standardtypsnitt, fontstorlekar och avstånd. Detta kan resultera i en monoton och visuellt ointressant webbplats. Genom att modifiera ett html-elements storlek, placering, färg och layout med CSS så kan vi förbättra en webbplats utseende, vilket resulterar i en mer lockande användarupplevelse.

Hur använder vi oss av CSS?

CSS kan implementeras genom tre olika metoder:

  • Separat CSS-fil
  • I dokumentet
  • Direkt i HTML-elementet

Separat CSS-fil - Det vanligaste sättet att implementera CSS på en webbsida är genom att använda en separat CSS-fil. CSS-filen kopplas till dokumentet genom HTML-elementet <link> i dokumentets head sektion. CSS-filen skapas på samma sätt som ett HTML-dokument i en vanlig textredigerare, och sparas med filändelsen .css. Ett vanligt namn på denna fil är style.css, men vilket namn som helst kan användas så länge .css används som filändelse. Flera CSS-filer kan användas, och samma CSS-fil kan användas av flera HTML-dokument.

I dokumentet - CSS-formateringen kan också användas genom att lägga till HTML-elementet <style> i HTML-dokumentets head sektion. När CSS-koden skrivs in i ett style-element gäller den endast för det specifika dokumentet där koden finns. Det innebär att CSS-formateringen inte kan återanvändas i andra dokument, utan måste skrivas om för varje separat dokument.

Direkt i HTML-elementet - Det sista sättet att använda CSS är att skriva det direkt i det element som ska formatera genom att använda attributet “style”. CSS-formateringen tillämpas då endast på det element där koden är skriven vilket innebär att när formatet ska användas i ett annat element så måste koden skrivas om på nytt. Detta gör att dokumenten innehåller mycket kod som tar lång tid att ladda och är därför det minst effektiva sättet att använda CSS-formatering.

Vilka versioner finns det?

Det finns tre olika versioner av CSS, CSS1, CSS2 samt CSS3. Var och en av versionerna har infört förbättringar och nya funktioner jämfört med dess föregångare.

CSS1 är den första versionen som släpptes i december 1996. Med CSS1 blev det möjligt att utförande grundläggande styling, som bland annat typsnitt egenskaper, textfärg, bakgrund och textattribut.

CSS2 släpptes i maj 1998 och utgjorde en förbättring jämfört med CSS1 genom att introducera nya funktioner som möjliggjorde för mer avancerade layouter, positionering och stöd för olika mediatyper.

CSS3 är den nyaste versionen av CSS som har utvecklats och uppdaterats över tid, med olika moduler som har införts och utvecklats efter behov. CSS3 har möjliggjort skapandet av ännu mer dynamiska och engagerande webbplatser genom layout tekniker som flexbox och grid, möjligheten att skapa effekter som skuggor och övergångar, samt möjliggjort för responsiv design.

Varför finns det olika versioner? (Varför är det inte bara en?)

Det finns flera skäl till varför det finns flera versioner av CSS istället för bara en. Den första är på grund av teknisk utveckling. Webben förändras och utvecklas kontinuerligt vilket leder till att nya behov uppkommer över tid. Därför behöver standarderna för CSS anpassas och utvecklas för att möta dessa förändringar. Genom att uppdatera och införa flera versioner av CSS kan dessa förändringar mötas.

En annan anledning är på grund av stöd för olika enheter och plattformar. Flera versioner av CSS gör det möjligt att standarderna anpassas för att ge bättre stöd för olika enheter, webbplatser och plattformar. Eftersom flera versioner av CSS existerar så kan äldre enheter, webbplatser och plattformar som är uppbyggda med en äldre version av CSS fungera som vanligt och behöver därför inte göras om när en ny version av CSS släpps.

Varför är det viktigt att veta vilken version som en viss CSS-egenskap tillhör? (Kan detta påverka webbplatsen?)

Det är viktigt för utvecklare att veta vilken version en viss CSS-egenskap tillhör eftersom det kan ha inverkan på hur en webbsida tolkas och visas på olika webbläsare och enheter. Om en webbplats använder en CSS-egenskap som inte alla användares webbläsare eller enheter stödjer kan det leda till att vissa delar av webbplatsen inte fungerar som den ska hos alla användare. Vissa CSS-egenskaper och funktioner kan vara olika beroende på vilken version av CSS som används och vilken webbläsare samt enhet användaren använder. Genom att vara medveten om vilka CSS-egenskaper som tillhör vilken version av CSS kan en mer enhetlig och tillförlitlig användarupplevelse säkerställas.
