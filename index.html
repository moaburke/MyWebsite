Att använda CSS

Vad är skillnaden mellan “inline” CSS och externa stilmallar?

CSS kan implementeras genom tre olika metoder:

  • Inline CSS
  • Intern stilmall
  • Extern stilmall

Inline CSS

Inline CSS används för att formatera ett specifikt element genom att använda attributet style direkt i HTML-elementet.

För att formattera ett specifikt p-element kan vi skriva följande:

<p style="color: red; font-size: 20px;">En röd paragraf</p>

I detta exempel tillämpas en röd färg och en textstorlek på 20px på p-elementet.

Fördel med Inline CSS

Inline CSS är användbart för snabba och specifika stiländringar på enskilda HTML-element. Dessa stiländringar tillämpas endast på det specifika HTML-elementet där koden är skriven. Inline CSS kan därför vara fördelaktikt för stiländringar som inte kommer att användas på andra delar av webbsidan.

Nackdel med Inline CSS

Eftersom inline CSS endast tillämpar stiländringar på specifika HTML-elementet blir det mindre effektivt när samma stiländringar används på flera delar av webbplatsen. Att formatera varje enskilt HTML-element blir inte bara tidskrävande, utan ökar även risken för fel. Dessutom blir koden svår att underhålla, och den ökade filstorleken på HTML-dokumentet kan påverk webbsidans prestanda negativt.

Intern stilmall

En intern stilmall är en stilmall som används för att tillämpa stiländringar i ett specifikt HTML-dokument. Stiländringarna definieras inom <style>-taggar i head-sektion av HTML-dokumentet.

För att formattera p-element likt exemplet ovan med intern stilmall kan vi skriva följande:

<!DOCTYPE html> <html> <head> <style> p { color: red; font-size: 20px; } </style> </head> <body> <p>En röd paragraf</p> </body> </html>
Fördel med Intern Stilmall

Att använda en intern stilmall är fördelaktikt för mindre projekt eller på enskilda sidor där behover av att skapa en separat CSS-fil är minimalt. Genom att definiera stilar internt kan stiländringar kan göras snabbt och hanteras enkelt, vilker sparar tid och minskar komplexiteten i projektet.

Nackdel med Intern Stilmall

Stilinformationen som tillämpas genom inline CSS är bunden till det specifika HTML-dokumen där det definieras och kan inte återanvändas mellan flera sidor. Detta kräver att stildefinitionerna upprepas i varje HTML-dokument, vilket kan göra underhåll och ändrig av kod mindre effektiv.

Extern stilmall

En extern stilmall använder en separat CSS-fil som innehåller all stilinformation. Denna CSS-fil länkas till dokumentet genom HTML-elementet <link> placerat i head-sektionen av varje HTML-dokument där stilinformationen ska tillämpas.

För att formattera p-element likt exemplet ovan med extern stilmall kan vi skriva följande:

index.html <!DOCTYPE html> <html> <head> <link rel="stylesheet" href="style.css"> </head> <body> <p>En röd paragraf</p> </body> </html>
style.css p { color: red; font-size: 20px; }
Fördel med Extern Stilmall

Att avända en extern stillmall är den vanligaste metoden för att impletera CSS. Genom att separera stilmallen från HTML-koden kan stilinformaitonen återanvändas över hela webbplatsen, vilket möjliggör enkel tillämpning och underhåll av ett enheltligt utseende. Genom att placera stilmallen i en separat fil minskas storleken på HTML-filen, vilket resulterar i snabbare sidoladdningar och förbättrad användaruppleveslse.

Nackdel med Extern Stilmall

Trots de många fördelarna med externa stilmallar finns det även några nackdelar. För det första innebär användningen av en extern stillmall ett ökat beroendeförhållande. Webbplatsens utseende är beroende av att länken till stilmallen är fungerande för att kunna tillämpa stilinformationen korrekt. Vid eventuella problem med länken kan webbplatsen visas utan stilinformaitonen, vilket kan resultera i en negativ användaruppleveslse. För det andra kan användning av externa stilmallar leda till ökad laddningstid varje gång webbplatsen laddar en sida. Detta beror på att varje gång en sida laddas måste stilmallen hämpas separat, vilket innebär en extra HTTP-begäran och därmed en ökning av sidans laddningstid.
